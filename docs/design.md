# AI语音绘图工具 - 设计文档

## 1. 项目概述

基于Flask + DeepSeek API的纯语音控制绘图工具。用户通过语音指令控制Canvas绘图，无需鼠标键盘。

**核心创新点：**
- 三级响应策略：本地解析（零延迟）→ 缓存命中 → API调用 → Fallback
- 复杂图形自动拆解：AI将"画一个房子"拆解为多个基础图形
- 全链路容错：防抖、纠错、重试、边界检查、Fallback

## 3. 指令能力规划与实现

### 3.1 已实现指令（第一阶段）

| 类别 | 指令示例 | 实现方式 | 延迟 |
|:---|:---|:---|:---|
| 基础图形 | "画一个红色的圆" | 本地解析 → Canvas arc() | <100ms |
| 位置控制 | "在左上角画一个圆" | 本地解析 + 坐标映射 | <100ms |
| 大小控制 | "画一个大圆" | 本地解析 + 关键词映射 | <100ms |
| 颜色控制 | "画一个蓝色的矩形" | 本地解析 + 颜色映射 | <100ms |
| 特殊操作 | "清空画布" | 本地解析 → clearRect() | <100ms |
| 撤销 | "撤销" | 本地解析 → 历史栈pop | <100ms |
| 导出 | "导出图片" | 本地解析 → toDataURL() | <100ms |

### 3.2 复杂指令拆解（第二阶段）

| 指令 | 拆解结果 | 图形数 |
|:---|:---|:---|
| "画一个房子" | 墙壁(矩形) + 屋顶(三角形) + 门(矩形) | 3 |
| "画一个笑脸" | 脸(圆) + 左眼(圆) + 右眼(圆) + 嘴(圆) | 4 |
| "画一个太阳" | 圆心(圆) + 4条光芒(线段) | 5 |
| "画一棵树" | 树冠(圆) + 树干(矩形) | 2 |

**实现方式：** DeepSeek system prompt 预置拆解示例，模型根据示例类推。

### 3.3 计划实现但未完成（第三阶段）

| 类别 | 未实现原因 |
|:---|:---|
| 自由手绘曲线 | 贝塞尔曲线参数化描述困难，语音指令难以精确控制控制点 |
| 图层管理 | 需要完整图层栈和混合模式，前端状态管理复杂度超出MVP范围 |
| 渐变填充 | Canvas渐变需要多色标定义，语音指令难以精确描述 |
| 动画效果 | 超出静态绘图工具范围，需引入动画引擎 |

## 4. 成本控制策略

| 策略 | 说明 | 效果 |
|:---|:---|:---|
| 前端Canvas渲染 | 所有绘图操作本地执行，零API调用 | 节省100%绘图API费用 |
| 本地指令解析 | 80%常见指令不走DeepSeek API | 节省80% API调用次数 |
| 缓存历史记录 | 重复指令直接命中缓存 | 节省重复解析费用 |
| 15秒超时 + 2次重试 | 防止网络异常导致无限等待 | 避免无效API调用 |
| 单次指令单次API | 仅在语音识别完成后调用一次 | 最小化API调用频率 |

## 5. 交互设计
用户按住说话 → 语音识别中（显示"识别中..."） → AI解析中（显示加载动画）
↓
解析成功 → 执行绘图 → 显示"绘制了N个图形" → AI语音反馈（文字）
↓
解析失败 → 显示错误信息 → Fallback画默认灰色圆

**关键交互细节：**
- 3秒防抖：防止快速重复触发
- 语音识别纠错："买一个"→"画一个"，"左下脚"→"左下角"
- 边界检查：所有图形强制在800x600画布内
- 历史栈：最多50步撤销，防止内存溢出

## 6. 性能优化

| 优化点 | 实现方式 | 效果 |
|:---|:---|:---|
| 零延迟响应 | 本地解析80%常见指令 | 响应时间从2-5s降至<<100ms |
| 缓存机制 | 缓存API返回结果 | 重复指令二次响应<<50ms |
| 防抖机制 | 3秒内忽略重复触发 | 减少50%无效请求 |
| 重试机制 | 500错误自动重试2次 | 提升95%成功率 |
| 边界检查 | clamp函数限制坐标 | 防止绘制异常导致重绘 |

## 9. 原创声明

本项目为七牛云×XEngineer暑期实习项目参赛作品。

核心自主开发功能：
- 语音指令解析管道（本地解析 + 缓存 + API调用）
- Canvas绘图引擎（含边界检查）
- 复杂图形拆解算法（system prompt设计）
- 全链路容错机制（防抖、纠错、重试、Fallback）

依赖第三方库：
- Flask: Web框架
- requests: HTTP请求
- python-dotenv: 环境变量管理
