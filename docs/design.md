# AI语音绘图工具 - 设计文档

## 1. 项目概述

基于Flask + DeepSeek API的纯语音控制绘图工具。用户通过语音指令控制Canvas绘图，无需鼠标键盘。

## 2. 技术架构
┌─────────────┐     ┌──────────────┐     ┌─────────────┐
│  麦克风输入  │────→│ Web Speech   │────→│ DeepSeek    │
│  (浏览器)    │     │ API (免费)   │     │ API (指令解析)│
└─────────────┘     └──────────────┘     └──────┬──────┘
│
↓
┌─────────────┐
│ JSON指令     │
│ {action,    │
│  shapes}     │
└──────┬──────┘
│
↓
┌─────────────┐
│ Canvas绘图   │
│ (纯前端)     │
└─────────────┘

## 3. 指令能力规划与实现

### 3.1 已实现指令

| 类别 | 指令示例 | 实现方式 |
|:---|:---|:---|
| 基础图形 | "画一个红色的圆" | DeepSeek解析→Canvas arc() |
| 位置控制 | "在左上角画一个圆" | System Prompt预定义坐标映射 |
| 大小控制 | "画一个大圆" | 关键词映射到具体数值 |
| 特殊操作 | "清空画布" | 直接调用Canvas clearRect() |

### 3.2 计划实现但未完成

| 类别 | 未实现原因 |
|:---|:---|
| 复杂图形拆解（如"画房子"） | 需要多步图形组合和坐标计算，当前单次API调用难以保证组合精度 |
| 自由手绘曲线 | 贝塞尔曲线参数化描述困难，语音指令难以精确控制控制点 |
| 图层管理 | 需要引入完整图层栈，前端状态管理复杂度超出MVP范围 |

## 4. 成本控制策略

| 策略 | 说明 |
|:---|:---|
| 前端Canvas渲染 | 所有绘图操作本地执行，零API调用 |
| 单次指令单次API | 仅在语音识别完成后调用一次DeepSeek |
| 缓存历史记录 | 撤销操作本地维护历史栈，不重复调用API |
| 15秒超时 | 防止网络异常导致无限等待 |

## 5. 交互设计

- 按住说话 → 识别中 → AI解析 → 执行绘图 → 语音反馈
- 指令日志实时显示，便于用户确认
- 快捷按钮作为语音的备用操作（符合"纯语音"要求，但提供辅助）
